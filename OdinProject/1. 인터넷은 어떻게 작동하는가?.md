###  [인터넷은 어떻게 동작하는가?](https://developer.mozilla.org/ko/docs/Learn/Common_questions/Web_mechanics/How_does_the_Internet_work)

###### 기본적인 컴퓨터와 컴퓨터간의 네트워킹은 물리적 케이블로 연결, 컴퓨터 대수가 많아질수록 이러한 연결은 복잡해지므로 **_라우터_**라는 특수 소형 컴퓨터를 통해 여러대의 컴퓨터를 연결하여 a라는 컴퓨터에서 정보를 보낼 때, b 컴퓨터에 제대로 전달되는지 확인하는 역할

###### 컴퓨터의 수가 더 많아지면 라우터와 라우터간의 연결을 통하여 확장할 수 있다 하지만 멀리 떨어진 지역과의 물리적인 연결은 힘듬 -> 전화기 기반 시설을 이용하여 해결

###### **_모뎀_**(네트워크 정보를 전화 기반 시설이 처리가능한 신호로 바꾸는 장치)

###### **_ISP(Internet Service provider)_**를 통해 네트워크와 네트워크를 타고 정보가 전달됨

####  인터넷과 웹의 차이

+ ###### **_인터넷_**은 수억대의 컴퓨터간의 네트워킹이 가능하게 하는 기술 인프라

+ ###### **_웹_**은 그 기반위에 구축된 서비스

----

#### IP 주소란

###### **_IP(Internet Protocol) Address_** : 각 컴퓨터마다 할당된 점으로 구분된 네 개의 숫자

###### 네트워크에 연결되는 모든 컴퓨터는 IP주소를 가지고 있다 <br>IP주소는 기억하기 어렵기 때문에 IP주소의 이름을 지정할 수 있는 도메인을 지정하여 보다 쉽게 이용 가능하다

---

### 클라이언트와 서버

######  **_클라이언트_**: 일반 사용자들의 인터넷이 연결된 장치 또는 웹에 접근하는 소프트웨어 (웹 브라우저)

###### **_서버_**: 웹페이지, 사이트, 앱을 저장하는 컴퓨터. 클라이언트가 웹페이지에 접근할 때, 서버로부터 클라이언트로 웹 페이지의 사본이 다운로드 됨

###### **_TCP/IP (Transmission Control Protocol / Internet Protocol)_**: 어떻게 데이터가 인터넷을 통해 여행해야하는지에 관한 규약

###### **_DNS (Domain Name System)_**: 일종의 웹사이트를 위한 주소록으로, 브라우저에 웹 주소를 입력했을 때, 브라우저는 DNS에서 웹사이트의 IP주소를 먼저 살펴보고 **_HTTP_** 메시지를 올바른 곳으로 보낸다.

###### **_HTTP (Hypertext Transfer Protocol)_**: 서버와 클라이언트가 소통을 할 수 있게 하는 언어를 정의하는 어플리케이션 규약

###### Component files: 웹사이트를 구성하는 파일들

> ###### 코드 파일: 기본적으로 HTML, CSS, JavaScript로 생성
>
> ###### Assets: 이미지, 음악, 비디오, PDF 등 웹사이트를 만드는 다른 것들

----

### 웹 브라우저에 주소를 입력했을 때 발생하는 일

1. ##### 브라우저는 DNS서버에서 웹사이트의 IP주소를 찾는다

2. ##### 브라우저가 서버에게 웹사이트 사본을 요청하는 HTTP 메시지를 보낸다. 이러한 데이터는 TCP/IP 연결을 통해 전송된다.

3. ##### HTTP메시지를 받은 서버는 클라이언트의 요청을 승인하고 "200 OK" 메시지를 전송한다. 그리고 웹사이트의 파일들을 데이터 **_패킷_**이라는 작은 덩어리로 브라우저의 전송한다.

4. ##### 브라우저는 패킷들을 하나의 웹 사이트로 조립하고, 사용자 화면에 보여준다.

---

### 웹페이지, 웹 서버, 웹브라우저, 검색엔진 사이의 차이점

1. ##### 웹페이지: 웹 브라우저에서 보여지는 문서

2. ##### 웹사이트: 웹 페이지들의 모음

3. ##### 웹 서버: 인터넷에 웹사이트를 호스팅하는 컴퓨터

4. ##### 검색엔진: 다른 웹 페이지들을 찾게 도와주는 웹사이트 (ex. 구글, 빙, 야후)
